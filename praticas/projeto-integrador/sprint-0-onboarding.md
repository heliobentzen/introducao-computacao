# Sprint 0 — Onboarding e Definição do Problema

> **Módulo de referência:** [01 — Pensamento Computacional](../../conteudo/01-fundamentos-pensamento-computacional.md)  
> **Duração:** Semanas 1–2  
> **Entrega:** documento de escopo no repositório da equipe

---

## Objetivo

Aplicar os 4 pilares do pensamento computacional para definir e delimitar o problema que o sistema vai resolver.

---

## Tarefas

### T1. Definição do problema (Decomposição)

Escreva o problema em **1 parágrafo** (5–8 linhas) respondendo:

- Quem sofre com o problema?
- Qual a dor principal?
- Qual o impacto de não resolver?

**Decomponha** o problema em pelo menos **6 subproblemas** com entrada, processamento e saída.

### T2. Personas (Reconhecimento de Padrões)

Defina **3 personas** detalhadas:

| Atributo | Persona 1 | Persona 2 | Persona 3 |
|----------|----------|----------|----------|
| Nome e idade | | | |
| Perfil (semestre, rotina) | | | |
| Dor principal | | | |
| Objetivo com o sistema | | | |
| Nível técnico | | | |

Identifique **padrões** entre as personas: que necessidades são comuns? Quais são únicas?

### T3. Funcionalidades iniciais (Abstração)

Liste **8–10 funcionalidades** para o sistema. Para cada uma:

- Descrição em 1 frase.
- Persona(s) beneficiada(s).
- Prioridade: essencial / desejável / futura.

### T4. Priorização (Algoritmo de decisão)

Organize as funcionalidades numa **matriz Impacto × Esforço** (2×2):

|  | Baixo esforço | Alto esforço |
|--|--------------|-------------|
| **Alto impacto** | Fazer primeiro | Planejar |
| **Baixo impacto** | Se sobrar tempo | Descartar |

Selecione as **top 5** funcionalidades para o MVP.

### T5. Setup do repositório

1. Criar repositório Git no GitHub.
2. Estrutura mínima: README, pasta `docs/`, pasta `prototipos/`.
3. Primeiro commit com a documentação desta sprint.
4. Cada membro faz pelo menos 1 commit.
