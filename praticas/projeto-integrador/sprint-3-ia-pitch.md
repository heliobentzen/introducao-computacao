# Sprint 3 — Inteligência Artificial e Apresentação Final

> **Módulos de referência:** [07 — Inteligência Artificial](../../conteudo/07-inteligencia-artificial.md) e [08 — IA Generativa](../../conteudo/08-ia-generativa-responsavel.md)  
> **Duração:** Semanas 13–16  
> **Entrega:** feature de IA + pitch técnico final

---

## Objetivo

Incorporar uma **funcionalidade de IA** ao projeto, preparar a **apresentação final** e consolidar o aprendizado do semestre.

---

## Tarefas

### T1. Feature de IA para o sistema

Proponha e documente **uma funcionalidade que use IA** no sistema de apoio acadêmico. Exemplos:

| Ideia | Tipo de IA | Complexidade |
|-------|-----------|-------------|
| Sugestão automática de prioridade baseada em padrão do aluno | ML Supervisionado | Média |
| Agrupamento de tarefas por área de estudo | ML Não Supervisionado | Média |
| Chatbot de dúvidas sobre a disciplina | IA Generativa (LLM) | Alta |
| Resumo automático de anotações de aula | IA Generativa | Média |
| Previsão de risco de atraso em tarefas | ML Supervisionado | Alta |

**Documentação obrigatória:**

1. **Descrição da feature** (1 parágrafo).
2. **Tipo de IA e justificativa** (por que essa abordagem?).
3. **Dados necessários:** que dados o sistema coletaria? De onde viriam? Qual o formato?
4. **Ciclo de ML aplicado:**
   - Dados → Pré-processamento → Treinamento → Modelo → Inferência → Avaliação.
   - Para cada etapa, descreva o que aconteceria no contexto do seu projeto.
5. **Limitações e riscos:**
   - O que pode dar errado?
   - Viés potencial nos dados?
   - Como mitigar?
6. **Classificação EU AI Act:** em que nível de risco essa feature se enquadra?

### T2. Uso de IA generativa no projeto

Documente **como a equipe usou IA generativa** durante todo o projeto:

| Sprint | Uso de IA | Ferramenta | FACT (1-5) | Verificação feita? |
|--------|----------|-----------|-----------|-------------------|
| 0 | Ex: brainstorm de funcionalidades | ChatGPT | 4 | Sim, cruzamos com referências |
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

**Para cada uso, responda:**

- O que a IA gerou foi aceito integralmente, editado ou descartado?
- Quanto tempo economizou (ou custou)?

### T3. Protótipo final

Evolua o protótipo da Sprint 2 incorporando:

- A feature de IA (pode ser simulada/mockada).
- Pelo menos **3 fluxos completos** navegáveis.
- Tela de "como funciona a IA" para transparência com o usuário.

### T4. Pitch técnico (5–8 minutos)

Prepare apresentação com a seguinte estrutura:

| Slide/Seção | Tempo | Conteúdo |
|------------|-------|---------|
| 1. Problema | 1 min | Que dor resolvemos? Para quem? |
| 2. Solução | 1 min | O que o sistema faz? Demo rápida do protótipo |
| 3. Arquitetura | 1.5 min | Stack técnica: SO, cloud, dados |
| 4. IA | 1.5 min | Feature de IA: o quê, como, limitações |
| 5. Números | 1 min | Custo estimado, SLA, métricas de sucesso |
| 6. Aprendizado | 1 min | O que aprendemos? O que faríamos diferente? |
| 7. Perguntas | 1 min | Q&A |

**Formato:** vídeo gravado ou apresentação ao vivo (definido pelo professor).

### T5. Retrospectiva do semestre

Cada membro produz **reflexão individual** (300–400 palavras):

1. **Conceito mais importante** que aprendi na disciplina e por quê.
2. **Habilidade técnica** que desenvolvi e evidência concreta.
3. **Maior dificuldade** e como superei.
4. **Uso de IA:** avalio que usei de forma produtiva ou dependente? Justifique.
5. **Próximo passo:** o que quero aprender a seguir com base no que vi neste semestre?

## Encerramento

Este projeto não é sobre entregar software perfeito. É sobre demonstrar que você:

- Sabe decompor problemas e pensar com estrutura.
- Entende a stack tecnológica de ponta a ponta (hardware → SO → cloud → IA).
- Usa ferramentas modernas com responsabilidade.
- Comunica decisões técnicas com clareza.

Se saiu do semestre mais preparado para tomar decisões técnicas informadas, o objetivo foi alcançado.
