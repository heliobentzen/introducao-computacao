# Práticas da Disciplina

Roteiros prontos para laboratório, casa e trabalho em equipe — todos alinhados com os módulos teóricos de `conteudo/`.

---

## Estrutura

```
praticas/
├── checkpoints/           ← Verificações curtas de aprendizagem (1 por par de módulos)
│   ├── checkpoint-01.md   → Módulos 01 + 02 (Pensamento Computacional + Lógica)
│   ├── checkpoint-02.md   → Módulos 03 + 04 (Hardware + Dados)
│   ├── checkpoint-03.md   → Módulos 05 + 06 (Sistemas Operacionais + Cloud)
│   └── checkpoint-04.md   → Módulos 07 + 08 (IA + IA Generativa)
├── desafios/              ← Problemas maiores para autonomia e criatividade
│   ├── desafio-01.md      → Modelagem computacional de processo real (Módulos 01-03)
│   ├── desafio-02.md      → Arquitetura cloud para negócio real (Módulos 04-06)
│   └── desafio-03.md      → Uso crítico de IA generativa (Módulos 07-08)
└── projeto-integrador/    ← Trilha incremental do mini projeto da turma
    ├── README.md
    ├── sprint-0-onboarding.md
    ├── sprint-1-modelagem.md
    ├── sprint-2-infraestrutura.md
    └── sprint-3-ia-pitch.md
```

---

## Fluxo de uso

```
Módulo teórico → Checkpoint da semana → Desafio → Sprint do projeto integrador
      ↑                                                      |
      └──────────── Feedback + reflexão ←────────────────────┘
```

1. **Leia o módulo teórico** correspondente em `conteudo/`.
2. **Execute o checkpoint** da semana para verificar entendimento.
3. **Avance para o desafio** quando sentir domínio do tema.
4. **Atualize o projeto integrador** aplicando os conceitos na sprint ativa.

---

## Formato de entrega (padrão para todas as atividades)

- **Repositório Git** com README da atividade.
- **Evidências:** capturas de tela, vídeo curto, saída de terminal ou código.
- **Reflexão final obrigatória:** "o que quebrou, como resolvi e o que aprendi".
- **Citação de IA:** se usou IA generativa, indique o modelo, o prompt e o que foi aceito/editado/descartado.

---

## Princípios pedagógicos

- Menos copiar e colar, mais explicar o raciocínio.
- Errar documentado vale mais que acertar sem entender.
- Iteração com feedback é o motor do aprendizado.
- IA é ferramenta de apoio, não de substituição.
